Help & FAQ

Find answers to common questions and understand how Aegis Cloud Scanner works.

How It Works: Trust & Transparency

How does Aegis scan my AWS account?

Aegis uses the official AWS SDK, Boto3, to connect to your AWS account with the credentials you provide. It makes **read-only** API calls to check for common misconfigurations across various services like S3, IAM, and EC2. **Aegis cannot and will not modify any of your AWS resources.** It is a passive scanning tool, designed to provide visibility into potential security issues without performing any destructive actions.

What data is collected and how is it used?

The application collects the following data to provide its core functionality:

  • AWS Credentials: Your AWS Access Key ID and Secret Access Key. The Secret Access Key is **immediately encrypted** using a unique, app-specific key and is never stored or transmitted in plaintext.
  • Scan Results: Details about the findings from your scans (e.g., service, resource name, issue, status). This data is stored locally in your database to power the dashboard and historical trend charts.
  • User Data: Your username and email for account management. Your password is never stored in plaintext, but rather as a secure hash using `Flask-Bcrypt`.
  • Audit Logs: Records of user actions, such as logins and password changes, to monitor for suspicious activity.
We do not have access to your AWS data, such as the contents of your S3 buckets or EC2 instance data. Your data remains on your machine and is never shared with us or any third parties.

How do you guarantee my data is safe?

Your security is our top priority. We implement the following measures to protect your information:

  • Strong Encryption: Your AWS Secret Access Key is encrypted at rest using `cryptography.fernet`, an industry-standard symmetric encryption library.
  • Password Hashing: User passwords are not stored directly, but as irreversible hashes using `Flask-Bcrypt`, protecting them from breaches.
  • Secure Communication: All web traffic is secured with HTTPS to prevent man-in-the-middle attacks.
  • Two-Factor Authentication: We provide mandatory 2FA to prevent unauthorized access to your account.

General Questions

What is an AWS misconfiguration?

It's a security flaw in the way an AWS resource is configured. For example, an S3 bucket that is unintentionally publicly accessible, or an IAM user with a weak password policy, can lead to security vulnerabilities.

What AWS permissions do I need to run a scan?

We recommend creating a dedicated IAM user with the following managed policies attached: `SecurityAudit` and `ReadOnlyAccess`. This gives the scanner the necessary permissions to inspect your resources without the ability to modify them.
